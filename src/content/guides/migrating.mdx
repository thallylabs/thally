---
title: Migrating to Thally
description: Move your existing documentation from Mintlify, Docusaurus, GitBook, or any Markdown-based platform to Thally in one command.
---

The `create-thally-docs migrate` command clones a GitHub docs repository and converts it into a running Thally project — no manual copying, no reformatting.

```bash
npx create-thally-docs migrate https://github.com/your-org/your-docs my-docs
```

---

## How it works

<Steps>

<Step title="Clone the source repo">

Thally does a shallow `git clone` of the repository to a temporary directory. No authentication is required for public repos.

</Step>

<Step title="Auto-detect the platform and structure">

Thally inspects the repo for platform config files to understand the existing navigation:

| File | Platform detected |
|------|-------------------|
| `docs.json` / `mint.json` | Mintlify |
| `docusaurus.config.js` + `sidebars.js` | Docusaurus |
| `book.yaml` | GitBook |
| `_meta.json` | Nextra |
| `vitepress.config.*` | VitePress |
| `.starlight/` | Starlight |
| *(none of the above)* | Directory structure used |

The docs directory is also auto-detected: `docs/`, `documentation/`, `content/`, `src/content/`, `pages/`, or the repo root.

</Step>

<Step title="Import all documentation files">

Every `.md` and `.mdx` file is read directly. Frontmatter (`title`, `description`, `keywords`) is preserved. Platform-specific MDX components are converted to their Thally equivalents.

Non-Markdown files (e.g. `.rst`) are converted to MDX using Claude if you provide an Anthropic API key.

</Step>

<Step title="Wire up the API reference">

If an OpenAPI spec (`openapi.json`, `openapi.yaml`, `swagger.json`, etc.) is found anywhere in the repo, it is copied to `public/` and the API Reference tab in `docs.json` is set up automatically.

</Step>

<Step title="Write the Thally project">

All converted pages are written to `src/content/`. The `docs.json` navigation is built from the detected platform config or inferred from the directory structure.

</Step>

</Steps>

## Supported platforms

### Mintlify

```bash
npx create-thally-docs migrate https://github.com/mintlify/starter my-docs
```

Navigation is read from `docs.json` (Mintlify v3) or `mint.json`. OpenAPI specs in the `api-reference/` directory are detected and wired up automatically.

### Docusaurus

```bash
npx create-thally-docs migrate https://github.com/facebook/docusaurus/tree/main/website/docs my-docs
```

Sidebar structure is parsed from `sidebars.js`. Docusaurus admonitions (`:::note`, `:::warning`, etc.) are converted to Thally callout components.

### GitBook

```bash
npx create-thally-docs migrate https://github.com/your-org/gitbook-docs my-docs
```

`SUMMARY.md` is used to infer navigation order. `{% hint %}` blocks are converted to the appropriate Thally callout component.

### Plain Markdown

Any repository with Markdown files works — navigation is built from the directory structure.

```bash
npx create-thally-docs migrate https://github.com/your-org/docs my-docs
```

## Migrating into an existing project

Use `--into` to add content from another repo into an existing Thally project without overwriting it. New tabs and groups are merged; existing content is not touched.

```bash
npx create-thally-docs migrate https://github.com/your-org/api-docs --into ./my-docs
```

---

## Options

| Flag | Description |
|------|-------------|
| `--into <dir>` | Merge into an existing project |
| `--branch <name>` | Override the auto-detected git branch |
| `--docs-dir <path>` | Override the auto-detected docs subdirectory |
| `--api-key <key>` | Anthropic API key (or set `ANTHROPIC_API_KEY`) for non-Markdown conversion |
| `--yes`, `-y` | Skip confirmation prompts |

## Via the MCP server

If you're using Claude Code or Claude Desktop, you can trigger a migration through chat:

```
Migrate the Mintlify starter repo into a new Thally project at ~/dev/my-docs
```

The `migrate_docs` MCP tool accepts the same options as the CLI. See [MCP Server](/guides/mcp-server) for setup instructions.

## After migration

Once the migration completes, start the dev server to review the output:

```bash
cd my-docs
npm run dev
```

Then run `create-thally-docs check` to catch any issues before publishing:

```bash
npx create-thally-docs check my-docs
```

Common post-migration tasks:
- Update `src/data/site.ts` with your logo and brand colors
- Replace placeholder images in `src/content/`
- Review the nav order in `docs.json`
- Set `THALLY_SITE_URL` before deploying
