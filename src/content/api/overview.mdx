The pages in this tab are generated automatically from an OpenAPI spec. Every
operation gets its own page with parameter tables, request and response schemas,
code samples, and an interactive **Try It** console.

## The example spec

This demo is wired to [JSONPlaceholder](https://jsonplaceholder.typicode.com) — a
free, live, public REST API that needs no API key. Because it's a real endpoint,
the **Try It** console returns real responses, so you can explore the feature
before swapping in your own API.

- **Base URL:** `https://jsonplaceholder.typicode.com`
- **Auth:** none — every request works as-is
- **Resources:** Posts, Comments, and Users

`GET` requests return real data; write requests (`POST`, `PUT`, `PATCH`, `DELETE`)
are accepted and echoed back but not persisted.

## Fetch the spec

The spec is served from the docs site itself, in both formats:

- YAML: [`/openapi.yaml`](/openapi.yaml)
- JSON: [`/openapi.json`](/openapi.json)

```bash
curl http://localhost:3040/openapi.yaml
curl http://localhost:3040/openapi.json
```

## Use your own API

Replace the example with your real API in three steps:

<Steps>

<Step title="Drop in your spec">
Put your OpenAPI 3.x file in the project root as `openapi.yaml` (or `openapi.json`).
</Step>

<Step title="Point docs.json at it">
The `api` block on the API Reference tab references the file and orders the sidebar by tag.

```json
{
  "tab": "API Reference",
  "api": {
    "source": "openapi.yaml",
    "tagsOrder": ["Posts", "Comments", "Users"]
  }
}
```
</Step>

<Step title="Run the dev server">
`npm run dev` regenerates the endpoint pages from your spec — no manual page authoring.
</Step>

</Steps>

See [API Reference Setup](/guides/api-reference-setup) for the full list of
configuration options, including per-operation overrides and custom grouping.
