## Steps

<Steps>

<Step title="Install Dox">

Run the CLI to scaffold a new project:

```bash
npx create-dox my-docs
```

You'll be prompted for a project name, description, and brand preset.

</Step>

<Step title="Add your content">

Create `.mdx` files in `src/content/` and register them in `docs.json`:

1. Write your page with frontmatter (`title`, `description`)
2. Add the page slug to a group in `docs.json`
3. The sidebar updates automatically

<Note type="info" title="Tip">
Use the `showcase` page to preview all available components as you write.
</Note>

</Step>

<Step title="Deploy">

Build and deploy to any platform that supports Next.js:

```bash
npm run build
```

- **Vercel** — zero-config deployment
- **Netlify** — add a `netlify.toml` with the Next.js plugin
- **Docker** — use the included `Dockerfile` or roll your own

</Step>

</Steps>

## Accordion

<Accordion title="What is Dox?">
Dox is an open-source documentation template built with Next.js, MDX, and Tailwind CSS. It gives you a production-ready docs site with zero vendor lock-in.
</Accordion>

<Accordion title="How do I add a new page?">
Create an `.mdx` file in `src/content/`, then add its slug to a group in `docs.json`. The sidebar updates automatically.
</Accordion>

<Accordion title="Can I use my own components?">
Yes. Create a component in `src/components/` and register it in `src/components/mdx/mdx-components.tsx`. It will be available in all MDX files.
</Accordion>
