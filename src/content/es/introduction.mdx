Thally es una plantilla de documentación de código abierto que te ofrece un sitio de documentación listo para producción desde el primer momento. Está construida con el App Router de Next.js, MDX para la redacción de contenido y Tailwind CSS para los estilos. Cada página que escribes es un archivo `.mdx` — sin CMS, sin dependencia de proveedores y con control total sobre tu infraestructura.

## Qué incluye

**Contenido y redacción**
- **Redacción de contenido en MDX** — escribe documentación en Markdown con componentes JSX como Steps, Tabs, Cards y Callouts
- **Navegación por pestañas** — organiza el contenido en pestañas, grupos y páginas mediante un único archivo `docs.json`
- **Generación de referencia de API** — añade una especificación OpenAPI y obtén documentación interactiva de endpoints con una consola "Pruébalo" integrada
- **Búsqueda de texto completo** — búsqueda instantánea en el cliente con puntuación de relevancia
- **Bloques de código** — resaltado de sintaxis con Shiki, grupos de código en múltiples lenguajes y copiar al portapapeles
- **Changelog con RSS** — página de changelog con un feed RSS en `/changelog/rss.xml`

**Herramientas**
- **CLI (`create-thally-docs`)** — crea un nuevo proyecto, migra desde otra plataforma o valida tu documentación con un solo comando
- **Servidor MCP (`@thallylabs/mcp`)** — 13 herramientas para Claude, Cursor y cualquier cliente MCP para crear, editar, buscar y migrar documentación mediante lenguaje natural
- **Migración** — importa desde Mintlify, Docusaurus, GitBook, Nextra y más con `create-thally-docs migrate`

**Funcionalidades de IA**
- **Widget de chat con IA** *(Thally Cloud)* — preguntas y respuestas flotantes impulsadas por Claude, basadas en el contenido real de tu documentación
- **`/llms.txt` y `/llms-full.txt`** — endpoints listos para LLM para que las herramientas de IA puedan indexar y leer tu documentación
- **`/api/docs/[slug]`** — páginas individuales como Markdown limpio para acceso programático

**Diseño e infraestructura**
- **Modo oscuro** — tematización automática clara/oscura con colores de marca personalizables
- **Diseño responsive** — barra lateral, migas de pan, tabla de contenidos y diseño adaptado para móviles
- **Imágenes OG dinámicas** — imágenes de vista previa social generadas automáticamente que coinciden con tu marca
- **SEO integrado** — sitemap.xml, robots.txt y metaetiquetas por página incluidos de serie
- **Anclas de encabezado** — enlaces `#` de clic para copiar en cada encabezado para compartir fácilmente
- **Selector de versión** — menú desplegable configurable para cambiar entre versiones de la documentación
- **Integración de analíticas** — soporte conectable para Google Analytics, Plausible o PostHog
- **Editar en GitHub** — enlaces automáticos de "Editar esta página" cuando configuras la URL de tu repositorio

## Primeros pasos

La forma más rápida de crear un nuevo proyecto Thally es con la CLI:

```bash
npx create-thally-docs my-docs
```

La CLI te solicita un nombre de proyecto, descripción, preajuste de marca y URL opcional del repositorio de GitHub — luego crea un sitio de documentación listo para ejecutar con contenido inicial, navegación y dependencias instaladas.

Alternativamente, clona la plantilla manualmente:

```bash
git clone https://github.com/thallylabs/thally.git my-docs
cd my-docs
npm install
npm run dev
```

## Cómo funciona

Thally utiliza un modelo de contenido basado en archivos. Toda la documentación reside en `src/content/` como archivos `.mdx`. La estructura de navegación se define en `docs.json` en la raíz del proyecto — allí listas tus pestañas, grupos y páginas, y la barra lateral, las migas de pan y el enrutamiento se derivan automáticamente.

Para la documentación de API, apunta la pestaña de Referencia de API en `docs.json` a un archivo de especificación OpenAPI. Thally analiza la especificación y genera páginas de endpoints con tablas de parámetros, esquemas de solicitud/respuesta, ejemplos de código y un playground interactivo.

## Cuándo usar Thally

Thally es una buena opción cuando quieres:

- **Publicar documentación rápido** — empieza a escribir contenido en minutos, no en días
- **Controlar tu stack** — sin dependencias de servicios de terceros ni tarifas mensuales
- **Personalizar libremente** — extiende la aplicación Next.js con tus propios componentes, analíticas o autenticación
- **Documentar APIs** — genera automáticamente documentación de referencia a partir de una especificación OpenAPI
