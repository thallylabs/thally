## Compilar para producción

```bash
npm run build
```

Esto genera una compilación de producción optimizada en el directorio `.next/`. Todas las páginas de documentación se pre-renderizan de forma estática para una carga rápida.

## Variables de entorno

Antes de desplegar, establece la variable de entorno `THALLY_SITE_URL` con tu dominio de producción. Se utiliza para los metadatos de OpenGraph y las URLs canónicas:

```bash
THALLY_SITE_URL=https://docs.yourproduct.com
```

## Analíticas y almacenamiento durable

En local, Thally guarda analíticas y estado del admin en archivos SQLite bajo `.data/` — sin configuración. En hosts **serverless** (por ejemplo Vercel), ese sistema de archivos no persiste: necesitas una base de datos libSQL remota (Turso) si quieres métricas de tráfico o ajustes del admin durables.

```bash
THALLY_ANALYTICS_DB_URL=libsql://tu-db.turso.io
THALLY_ANALYTICS_DB_TOKEN=…
THALLY_DATABASE_URL=libsql://tu-db.turso.io
THALLY_DATABASE_TOKEN=…
```

Pasos completos (incluido el CLI de Turso) en la guía en inglés: [Deploying → Analytics & durable storage](/guides/deploying#analytics--durable-storage). Agent Readiness no usa esta base de datos.

## Vercel

La forma más rápida de desplegar — sube a GitHub e importa el repositorio:

1. Sube tu repositorio de documentación a GitHub
2. Ve a [vercel.com/new](https://vercel.com/new) e importa el repositorio
3. Establece `THALLY_SITE_URL` en las variables de entorno
4. Si quieres analíticas del admin, también configura las variables de Turso de arriba
5. Despliega

Vercel detecta automáticamente Next.js y configura todo.

## Netlify

```bash
npm run build
```

Establece el comando de compilación como `npm run build` y el directorio de publicación como `.next`. Es posible que necesites el plugin `@netlify/plugin-nextjs` para soporte completo de Next.js.

## Docker

Crea un `Dockerfile`:

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./
COPY --from=builder /app/public ./public
COPY --from=builder /app/src/content ./src/content
COPY --from=builder /app/openapi.yaml ./
COPY --from=builder /app/docs.json ./

EXPOSE 3000
CMD ["npm", "start"]
```

Compila y ejecuta:

```bash
docker build -t my-docs .
docker run -p 3000:3000 my-docs
```

## Cloudflare Pages

Usa el adaptador `@cloudflare/next-on-pages`:

1. Instala: `npm install @cloudflare/next-on-pages`
2. Establece el comando de compilación: `npx @cloudflare/next-on-pages`
3. Establece el directorio de salida: `.vercel/output/static`
4. Despliega a través del panel de Cloudflare o la CLI de Wrangler

## Exportación estática

Si no necesitas funcionalidades del lado del servidor (como el proxy de Prueba), puedes exportar como un sitio completamente estático. Agrega a `next.config.ts`:

```ts
const nextConfig = {
  output: 'export',
}
```

Luego ejecuta `npm run build` — la salida estará en el directorio `out/`, lista para cualquier servidor de archivos estáticos.

<Note type="warning" title="Limitación de la exportación estática">
  El panel "Pruébalo" de la API requiere una ruta proxy del lado del servidor y no funcionará con la exportación estática. Todas las demás funcionalidades funcionan correctamente.
</Note>

## SEO y feeds

Una vez desplegado, Thally sirve automáticamente:

- **`/sitemap.xml`** — incluye todas las páginas de documentación, páginas de referencia de la API y el registro de cambios
- **`/robots.txt`** — permite a los rastreadores en todas las páginas, bloquea las rutas internas de la API (`/api/og`, `/api/try-it`)
- **`/changelog/rss.xml`** — feed RSS para las actualizaciones del registro de cambios
- **Imágenes OG dinámicas** — cada página obtiene una imagen de vista previa social personalizada a través de `/api/og`

Todos estos usan `THALLY_SITE_URL` para generar URLs absolutas, así que asegúrate de que esté configurado antes de desplegar.
