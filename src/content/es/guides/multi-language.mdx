Dox soporta documentación multi-idioma en tiempo de compilación. Los archivos MDX traducidos se confirman en tu repositorio, lo que te brinda SSG completo, costo cero en tiempo de ejecución y URLs realmente indexables para cada locale.

**Por qué traducciones en tiempo de compilación:**

- Los archivos MDX traducidos están bajo control de versiones — revisables y editables como cualquier otro contenido
- Cada locale obtiene URLs reales e indexables (p. ej. `/es/introduction`) — máximo valor SEO
- Funciona con cualquier hosting sin infraestructura edge
- Sin almacén KV, sin llamadas API en tiempo de ejecución, sin costo por solicitud

---

## Estructura de URL

- **Locale predeterminado** — `/introduction`, `/guides/getting-started` ← sin prefijo, totalmente compatible con versiones anteriores
- **Locales secundarios** — `/es/introduction`, `/es/guides/getting-started`

---

## Paso 1 — Configurar i18n en docs.json

Agrega un bloque `i18n` a tu `docs.json`:

```json
{
  "i18n": {
    "defaultLocale": "en",
    "locales": [
      { "code": "en", "label": "English" },
      { "code": "es", "label": "Español" },
      { "code": "fr", "label": "Français" }
    ]
  },
  "tabs": [...]
}
```

Una vez agregado esto, un **selector de idioma** (ícono de globo) aparecerá automáticamente en la barra superior.

---

## Paso 2 — Traducir páginas con la CLI

Ejecuta el comando de traducción para generar archivos MDX traducidos para un locale:

```bash
npx create-dox translate --locale es
```

Esto traduce **todas las páginas** y las escribe en `src/content/es/`.

### Traducir páginas específicas

```bash
npx create-dox translate --locale es --pages introduction,guides/getting-started
```

### Sobrescribir traducciones existentes

```bash
npx create-dox translate --locale es --force
```

### Todas las opciones

| Bandera | Descripción |
|---------|-------------|
| `--locale <code>` | Locale de destino — requerido (debe coincidir con un código en la configuración i18n de `docs.json`) |
| `--pages <ids>` | IDs de página separados por comas para traducir (omitir para todas las páginas) |
| `--force` | Sobrescribir archivos de traducción existentes |
| `--api-key <key>` | Clave API de Anthropic (usa `ANTHROPIC_API_KEY` como variable de entorno si no se especifica) |
| `--model <id>` | Modelo Claude a usar (predeterminado: `claude-sonnet-4-6`) |
| `--yes`, `-y` | Omitir la confirmación |

---

## Paso 3 — Estructura de archivos de contenido

Después de ejecutar `translate`, tu directorio de contenido se verá así:

```
src/content/
├── introduction.mdx          ← principal (inglés, sin cambios)
├── guides/
│   └── getting-started.mdx
├── es/                        ← traducciones al español
│   ├── introduction.mdx
│   └── guides/
│       └── getting-started.mdx
└── fr/
    └── introduction.mdx
```

Puedes editar los archivos traducidos igual que cualquier otro contenido MDX.

---

## Calidad de la traducción

El comando de traducción usa Claude para traducir tu documentación con estas reglas estrictas:

- Traduce todo el texto en prosa, encabezados y párrafos
- Traduce `title`, `description` y `keywords` del frontmatter
- Preserva todas las etiquetas de componentes MDX textualmente (`<Note>`, `<Steps>`, `<CodeGroup>`, etc.)
- Preserva todos los bloques de código e código en línea tal como están
- Preserva todas las URLs y rutas de archivos

---

## Comportamiento de reserva

Si una página no ha sido traducida para un locale, Dox sirve el contenido en el idioma principal y muestra un banner ámbar:

> ⚠️ Esta página aún no ha sido traducida a **es**. Mostrando la versión original en **EN**.

Si el archivo principal ha sido actualizado después de que se creó la traducción, Dox muestra un banner amarillo de antigüedad:

> Esta traducción puede estar desactualizada. [Ver original →]

Ambos banners solo aparecen en rutas de locales secundarios — la ruta principal nunca se ve afectada.

---

## Selector de idioma

Una vez que tienes 2 o más locales configurados, aparece un **ícono de globo** en la barra superior. Al hacer clic, se abre un menú desplegable con todos los locales configurados. Seleccionar un locale navega a la misma página en ese locale.

---

## SEO

Dox agrega automáticamente etiquetas `hreflang` y `canonical` para todos los locales configurados:

```html
<link rel="canonical" href="https://example.com/introduction" />
<link rel="alternate" hreflang="en" href="https://example.com/introduction" />
<link rel="alternate" hreflang="es" href="https://example.com/es/introduction" />
<link rel="alternate" hreflang="fr" href="https://example.com/fr/introduction" />
```

La URL principal siempre es la canónica, incluso para páginas de locales secundarios.

## Usar la herramienta MCP

Si usas el servidor MCP de Dox, puedes traducir documentación directamente desde tu asistente de IA:

```json
{
  "name": "translate_docs",
  "arguments": {
    "projectDir": "/path/to/my-docs",
    "locale": "es",
    "apiKey": "sk-ant-..."
  }
}
```

Consulta [Servidor MCP](/guides/mcp-server) para instrucciones de configuración.

---

## Inicializar con i18n

Al inicializar un nuevo proyecto, puedes habilitar el soporte multi-idioma desde el principio:

```bash
npx create-dox my-docs
```

Cuando se te pregunte **"¿Habilitar soporte multi-idioma?"**, ingresa `y` y proporciona tus códigos de locale (p. ej. `es,fr,de`). El bloque `i18n` se escribirá en `docs.json` automáticamente.
