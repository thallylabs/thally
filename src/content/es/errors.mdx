---
title: Estructura del proyecto
description: Archivos y directorios clave en un proyecto Dox.
---

Un proyecto Dox sigue la estructura estándar del App Router de Next.js con algunas convenciones para el contenido de documentación.

## Descripción general de directorios

| Ruta | Propósito |
| --- | --- |
| `docs.json` | Estructura de navegación: pestañas, grupos y páginas |
| `openapi.yaml` | Especificación OpenAPI para la referencia de API autogenerada |
| `src/content/` | Páginas de documentación MDX |
| `src/data/site.ts` | Nombre del sitio, descripción, colores de marca, analíticas y versiones |
| `src/components/` | Componentes de UI (diseño, MDX, referencia de API) |
| `src/app/` | Páginas y layouts del App Router de Next.js |
| `src/app/sitemap.ts` | sitemap.xml autogenerado |
| `src/app/robots.ts` | robots.txt autogenerado |
| `src/app/api/og/` | Endpoint de generación dinámica de imágenes OG |
| `src/app/changelog/rss.xml/` | Feed RSS para el registro de cambios |

## Archivos de contenido

Cada archivo `.mdx` en `src/content/` se convierte en una página de documentación. La ruta del archivo determina el slug:

- `src/content/introduction.mdx` → `/introduction`
- `src/content/guides/setup.mdx` → `/guides/setup`

## Archivos de configuración

- **`docs.json`** — define toda la navegación (pestañas, grupos de la barra lateral, orden de páginas)
- **`src/data/site.ts`** — metadatos del sitio, ajustes preestablecidos de marca y enlaces externos
- **`.env.local`** — variables de entorno como `DOX_SITE_URL`
