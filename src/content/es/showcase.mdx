## Pasos

<Steps>

<Step title="Instalar Dox">

Ejecuta el CLI para crear un nuevo proyecto:

```bash
npx create-dox my-docs
```

Se te pedirá un nombre de proyecto, descripción y un preset de marca.

</Step>

<Step title="Agrega tu contenido">

Crea archivos `.mdx` en `src/content/` y regístralos en `docs.json`:

1. Escribe tu página con frontmatter (`title`, `description`)
2. Agrega el slug de la página a un grupo en `docs.json`
3. La barra lateral se actualiza automáticamente

<Note type="info" title="Tip">
Usa la página `showcase` para previsualizar todos los componentes disponibles mientras escribes.
</Note>

</Step>

<Step title="Desplegar">

Compila y despliega en cualquier plataforma que sea compatible con Next.js:

```bash
npm run build
```

- **Vercel** — despliegue sin configuración
- **Netlify** — agrega un `netlify.toml` con el plugin de Next.js
- **Docker** — usa el `Dockerfile` incluido o crea el tuyo propio

</Step>

</Steps>

## Acordeón

<Accordion title="¿Qué es Dox?">
Dox es una plantilla de documentación de código abierto construida con Next.js, MDX y Tailwind CSS. Te proporciona un sitio de documentación listo para producción sin dependencia de proveedores.
</Accordion>

<Accordion title="¿Cómo agrego una nueva página?">
Crea un archivo `.mdx` en `src/content/`, luego agrega su slug a un grupo en `docs.json`. La barra lateral se actualiza automáticamente.
</Accordion>

<Accordion title="¿Puedo usar mis propios componentes?">
Sí. Crea un componente en `src/components/` y regístralo en `src/components/mdx/mdx-components.tsx`. Estará disponible en todos los archivos MDX.
</Accordion>
