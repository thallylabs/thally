---
title: "Columns"
description: "Place content side by side in a responsive multi-column layout."
---

`<Columns>` creates a CSS grid with a configurable number of columns. Unlike `<CardGroup>`, there is no card border — it is purely a layout primitive that collapses to a single column on mobile.

## Two columns

<Columns cols={2}>
  <div>
    **Left column**

    Use this for comparison tables, side-by-side code snippets, or multi-panel diagrams. Both columns stack vertically on mobile.
  </div>
  <div>
    **Right column**

    Each column can contain any MDX content — paragraphs, lists, code blocks, callouts, or nested components.
  </div>
</Columns>

```mdx
<Columns cols={2}>
  <div>Left content</div>
  <div>Right content</div>
</Columns>
```

## Combining with other components

Columns work well with `<Frame>` for side-by-side image comparisons:

<Columns cols={2}>
  <Frame caption="Before">
    <img src="/images/image1.jpg" alt="Before" />
  </Frame>
  <Frame caption="After">
    <img src="/images/image2.jpg" alt="After" />
  </Frame>
</Columns>

```mdx
<Columns cols={2}>
  <Frame caption="Before">
    <img src="/images/before.png" alt="Before" />
  </Frame>
  <Frame caption="After">
    <img src="/images/after.png" alt="After" />
  </Frame>
</Columns>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `cols` | `1 \| 2 \| 3 \| 4` | `2` | Number of columns |
